# Matrix-AVRO v0.3.0 Plan

## Scope

This plan addresses the module review findings and adds usability improvements for schema inspection, option ergonomics, decimal-safe writing, explicit schema declarations, and documentation. Each numbered section is intended to be implementable and reviewable as a separate pull request.

A task is only complete when its tests have passed. Run `./gradlew :matrix-avro:test` and `./gradlew :matrix-avro:codenarcMain :matrix-avro:codenarcTest` before marking any task done; record the exact commands and outcomes in the PR description. Final verification commands are in section 11.

## 1. Fix Negative Local Timestamp Reads

1.1 [x] Update `MatrixAvroReader.toLocalDateTimeMillis` to use `Math.floorMod(ms, MILLIS_PER_SECOND)` when calculating nanoseconds, matching the micros path and supporting values before the Unix epoch.

1.2 [x] Add tests with `local-timestamp-millis` and `local-timestamp-micros` values before `1970-01-01T00:00:00` to verify both paths decode correctly.

## 2. Fix Timezone-Sensitive LocalDateTime Writes

2.1 [x] In `MatrixAvroWriter.toTimestampMillisAvroValue`, `LocalDateTime` is currently converted via `ZoneOffset.systemDefault()`, which causes the same value to write differently depending on the JVM's default timezone. Chosen semantic: convert using `ZoneOffset.UTC` consistently.

2.2 [x] Update the write path to implement the chosen semantic. If `ZoneOffset.UTC` is chosen, verify the round-trip path in `MatrixAvroReader.toInstantMillis` is consistent.

2.3 [x] Add tests showing the same `LocalDateTime` value produces identical Avro bytes regardless of JVM timezone, or tests documenting the caller's responsibility for timezone-aware types.

## 3. Correct Stream Ownership Semantics

3.1 [x] Decide and document the stream ownership contract for `MatrixAvroReader.read(InputStream, ...)` and `MatrixAvroWriter.write(Matrix, OutputStream, ...)`: caller-owned streams remain open.

3.2 [x] If caller-owned streams should remain open, add close-shield wrappers in `MatrixAvroReader` and `MatrixAvroWriter` so closing `DataFileStream` or `DataFileWriter` does not close the caller's stream.

3.3 [x] Not applicable; streams are caller-owned and remain open.

3.4 [x] Add tests in `matrix-avro/src/test/groovy/test/alipsa/matrix/avro/MatrixAvroReaderTest.groovy` and `MatrixAvroWriterTest.groovy` proving the final stream ownership behavior for direct and options-based overloads.

## 4. Remove or Harden Schema Caching

4.1 [x] Reevaluate the schema cache in `MatrixAvroWriter`: removed it because schema inference depends on mutable Matrix data values beyond the current cache key, and the simpler uncached path passed module and full repository verification.

4.2 [x] Not applicable; the cache was removed.

4.3 [x] Add tests proving repeated schema builds of the same mutated `Matrix` instance produce fresh schemas.

4.4 [x] Not applicable; caching was not retained.

## 5. Align Public API Validation

5.1 [x] Update `MatrixAvroWriter.buildSchema(Matrix, boolean)` to call the same Matrix validation used by write paths, including null, empty matrix, and column-size checks.

5.2 [x] Update `MatrixAvroWriter.buildSchema(Matrix, AvroWriteOptions)` to fail with `IllegalArgumentException('Options cannot be null')` or the module's established validation style when options are null.

5.3 [x] Audit public reader and writer methods for inconsistent null handling or NPE-prone paths, and align them with the module's explicit validation exceptions.

5.4 [x] Add tests covering public `buildSchema` validation and any newly aligned validation paths.

## 6. Add Schema Inspection APIs

6.1 [x] Add `MatrixAvroReader.schema(File)`, `schema(Path)`, `schema(URL)`, `schema(byte[])`, and `schema(InputStream)` methods that return the Avro writer schema without reading all rows.

6.2 [x] Add options-aware schema inspection overloads where the reader schema can differ from the writer schema: specifically, `schema(File, AvroReadOptions)` and equivalent source variants that return the effective schema when `AvroReadOptions.readerSchema(...)` is set.

6.3 [x] Ensure schema inspection methods follow the same validation, naming, and stream ownership contract chosen in section 3.

6.4 [x] Add tests for schema inspection from file, path, byte array, and stream sources.

6.5 [x] Document schema inspection in `matrix-avro/README.md`, the Avro tutorial, and the Avro cookbook.

## 7. Improve Options Ergonomics

7.1 [x] Add static factory methods to `AvroReadOptions` and `AvroWriteOptions` for common starting points, such as `AvroWriteOptions.defaults()`, `AvroWriteOptions.exactDecimals()`, and `AvroReadOptions.named(String)`.

7.2 [x] Evaluated a closure-based Groovy configuration API and did not implement it; the existing typed fluent API plus factories keeps the surface statically typed and concise.

7.3 [x] Add convenience methods for common schema declarations, such as `AvroSchemaDecl.arrayOf(Class<?>)`, `mapOf(Class<?>)`, `decimalColumn(int, int)` naming alternatives, or similar helpers that reduce nested calls without introducing `Object`-typed APIs.

7.4 [x] Add tests proving the new factories and convenience helpers produce the expected options and Avro schemas.

7.5 [x] Document the new ergonomic API in README, tutorial, and cookbook examples without removing the existing direct and SPI examples.

## 8. Make Decimal-Safe Writes Easier

8.1 [x] Add a clearly named shortcut for decimal-safe writes, such as `MatrixAvroWriter.writeExactDecimals(...)`, `MatrixAvroWriter.writeDecimalSafe(...)`, or an `AvroWriteOptions.exactDecimals()` factory, while preserving existing overloads for compatibility.

8.2 [x] Default BigDecimal behavior remains unchanged for compatibility; decimal-safe shortcuts are documented prominently.

8.3 [x] Add tests showing BigDecimal columns round-trip as `BigDecimal` through the new decimal-safe shortcut and still follow the documented compatibility path through existing defaults.

8.4 [x] Update docs to explain the precision tradeoff in one place, with examples for default, decimal-safe, and explicit precision/scale writes.

## 9. Strengthen Explicit Schema Declaration Usability

9.1 [x] Add ergonomic aliases to `AvroSchemaDecl` for common nested declarations while keeping the existing typed declaration classes package-scoped.

9.2 [x] Expanded explicit schema declaration tests for the new aliases and existing nested array/map/record coverage; aliases are method conveniences and do not change SPI map serialization.

9.3 [x] Reviewed error messages from `AvroSchemaDecl.fromMap(...)` and `AvroWriteOptions.columnSchema(...)`; existing messages already name the expected typed alternatives.

9.4 [x] Document map-vs-record inference prominently, including how to force map encoding and record encoding with `columnSchema`.
