# Matrix-AVRO v0.2.0 Plan

## Scope

This plan breaks the `matrix-avro` usability, configurability, and discoverability follow-up work into PR-sized sections. Each numbered section is intended to be implementable and reviewable as a separate pull request.

A task is only complete when its tests have passed and the exact test command has been recorded in the section or PR description.

## 1. Reader Option Correctness and Naming Semantics

1.1 [x] Resolve the `lenientTypeConversion` contract in `matrix-avro/src/main/groovy/se/alipsa/matrix/avro/AvroReadOptions.groovy` and `matrix-avro/src/main/groovy/se/alipsa/matrix/avro/MatrixAvroReader.groovy`: either implement the documented behavior and thread it through all typed read overloads, or remove the option from the public API, SPI descriptors, docs, and tests if it is not being delivered in v0.2.0.

1.2 [x] Define and implement explicit read naming precedence in `MatrixAvroReader`: `AvroReadOptions.matrixName(...)` override first, otherwise the Avro record name from the file schema, otherwise the source-derived fallback such as the file name or `AvroMatrix`.

1.3 [x] Ensure `AvroReadOptions.toMap()` and `AvroReadOptions.fromMap(...)` only expose and round-trip read options that are actually supported by the reader implementation, with no dead configuration surface left behind.

1.4 [x] Add tests in `matrix-avro/src/test/groovy/test/alipsa/matrix/avro/MatrixAvroReaderTest.groovy` and `matrix-avro/src/test/groovy/test/alipsa/matrix/avro/AvroFormatProviderTest.groovy` for read naming precedence, `readerSchema(...)` behavior, read-option map round-tripping, SPI option parsing, and the chosen outcome for `lenientTypeConversion`.

1.5 [x] Update `matrix-avro/README.md`, the Avro tutorial page under the root `docs/tutorial/` section, and the Avro cookbook page under the root `docs/cookbook/` section to document the read defaults, the naming precedence, and the final supported read options without mentioning non-functional configuration.

## 2. Writer Defaults and Option Validation

2.1 [ ] Update `matrix-avro/src/main/groovy/se/alipsa/matrix/avro/AvroWriteOptions.groovy` and `matrix-avro/src/main/groovy/se/alipsa/matrix/avro/MatrixAvroWriter.groovy` so the default schema name comes from `matrix.matrixName` when present, falling back to `MatrixSchema` only when no Matrix name is available.

2.2 [ ] Add fail-fast validation for `compressionLevel` and `syncInterval` in `AvroWriteOptions` and `AvroWriteOptions.fromMap(...)`, including codec-specific level checks and clear `IllegalArgumentException` messages for invalid values.

2.3 [ ] Ensure `AvroWriteOptions.toMap()` and `AvroWriteOptions.fromMap(...)` preserve the validated write configuration without leaking ambiguous or partially-applied defaults.

2.4 [ ] Ensure the convenience boolean overloads and the typed `AvroWriteOptions` overloads produce the same schema naming defaults and validation behavior instead of drifting apart.

2.5 [ ] Add tests in `matrix-avro/src/test/groovy/test/alipsa/matrix/avro/MatrixAvroWriterTest.groovy` and `matrix-avro/src/test/groovy/test/alipsa/matrix/avro/AvroFormatProviderTest.groovy` for schema-name defaulting, invalid compression settings, invalid sync interval values, write-option map round-tripping, and direct-versus-SPI parity.

2.6 [ ] Update `matrix-avro/README.md`, the Avro tutorial page under the root `docs/tutorial/` section, and the Avro cookbook page under the root `docs/cookbook/` section to document the write defaults, especially schema naming, decimal inference defaults, and the validation behavior for compression-related options.

2.7 [ ] Run tests and record the command: `./gradlew :matrix-avro:test`

## 3. Explicit Per-Column Schema Control

3.1 [ ] Expand `matrix-avro/src/main/groovy/se/alipsa/matrix/avro/AvroWriteOptions.groovy` with typed per-column schema controls instead of relying only on heuristics. This should cover at least decimal precision/scale overrides and nested structure overrides for arrays, maps, and record-like columns.

3.2 [ ] Introduce typed declarations for per-column Avro schema control in `matrix-avro/src/main/groovy/se/alipsa/matrix/avro/` rather than `Object`-based options, so callers can force behavior such as map-vs-record encoding, array element types, map value types, and fixed decimal metadata with IDE support.

3.3 [ ] Refactor `matrix-avro/src/main/groovy/se/alipsa/matrix/avro/MatrixAvroWriter.groovy` so schema generation consults explicit per-column declarations first and only falls back to sampling heuristics when no override is provided.

3.4 [ ] Preserve the current heuristic behavior as the default path, but validate conflicting or incomplete per-column schema options with clear fail-fast errors.

3.5 [ ] Add tests in `matrix-avro/src/test/groovy/test/alipsa/matrix/avro/MatrixAvroWriterTest.groovy` and `matrix-avro/src/test/groovy/test/alipsa/matrix/avro/MatrixAvroRoundTripTest.groovy` for forced map vs record encoding, explicit array element typing, explicit decimal precision/scale, and conflicting option combinations.

3.6 [ ] Update `matrix-avro/README.md`, the Avro tutorial page under the root `docs/tutorial/` section, and the Avro cookbook page under the root `docs/cookbook/` section with direct API and SPI examples showing how to override nested schema inference explicitly.

3.7 [ ] Run tests and record the command: `./gradlew :matrix-avro:test`

## 4. Options-First API Ergonomics and Discoverability

4.3 [ ] Create or expand a dedicated Avro tutorial page under the root `docs/tutorial/` section covering options-first read/write usage, schema evolution, explicit schema control, and SPI round-tripping.

4.4 [ ] Create or expand a dedicated Avro cookbook page under the root `docs/cookbook/` section with focused recipes for common tasks such as decimal-safe writes, custom schema naming, schema evolution with `readerSchema(...)`, and controlling nested map/record behavior.
