# Matrix-Smile v0.2.0 Roadmap

## Findings from code review (2026-05-10)

### P1 — Bugs (blockers)

- [ ] **`tTestTwoSample(equalVariance)` is a no-op** — `SmileStats.groovy:306-310`
  Both branches call identical `TTest.test(x, y)`. If Smile exposes a pooled-variance overload, wire it up; otherwise remove the parameter and document that Welch's test is always used.

- [ ] **`MinMaxScaler.transform()` is missing** — `SmileFeatures.groovy:695-782`
  `StandardScaler` has `fit()`, `transform()`, and `fitTransform()`. `MinMaxScaler` only has `fit()` and `fitTransform()`. Any caller using fit-on-train / transform-on-test will fail at runtime. Add the missing `transform()` method.

- [ ] **`release.md` documents non-existent features** — `release.md` (v0.1.0)
  The following are claimed but do not exist in the codebase:
  classifiers (Logistic Regression, Gradient Boosted Trees, SVM, KNN, Naive Bayes, AdaBoost),
  clustering (Hierarchical, DENCLUE, CLARANS),
  dimensionality reduction (MDS, t-SNE),
  feature scaling (MaxAbsScaler, RobustScaler),
  imputation (mode), feature selection (sum/diff/product/ratio),
  distribution (Hypergeometric), "274 tests".
  Rewrite the v0.1.0 entry to reflect only what actually ships.

### P2 — DRY violations

- [ ] **`roundTo4()` duplicated in 3 classes** — `SmileClassifier.groovy:247`, `SmileRegression.groovy:308`, `SmileDimensionality.groovy:272`
  `SmileUtil.round(value, int)` already exists at `SmileUtil.groovy:136`. Remove the three private copies and call `SmileUtil.round(value, 4)`.

- [ ] **`matrixToArray()` duplicated in 2 classes** — `SmileCluster.groovy:248`, `SmileDimensionality.groovy:237`
  Move to `SmileUtil` as a shared static method and delegate from both classes.

- [ ] **`getNumericColumnNames()` duplicated in 3 classes** — `SmileStats.groovy:758`, `SmileFeatures.groovy` (local copy)
  Both should delegate to `SmileUtil.getNumericColumnNames(matrix)` and delete their local copies.

- [ ] **`correlationMatrix()` and `pValueMatrix()` duplicate `correlationWithSignificance()` internals** — `SmileStats.groovy:520`, `SmileStats.groovy:559`
  Since `correlationWithSignificance()` (line 598) already computes both, these two methods should just call it and extract the relevant key:
  ```groovy
  static Matrix correlationMatrix(Matrix matrix, ...) {
    correlationWithSignificance(matrix, columns, method)['correlation']
  }
  ```

- [ ] **`StandardScaler.fitTransform()` duplicates `transform()` loop** — `SmileFeatures.groovy:646-674`
  Once `MinMaxScaler.transform()` is added (see P1), both scalers' `fitTransform()` should call `fit()` then `transform()` instead of re-implementing the loop.

### P3 — Style / API

- [ ] **`SmileUtil.round()` should not be public API** — `SmileUtil.groovy:136`
  It is an internal helper with no GroovyDoc. Mark it private or move it to a non-public utility.

- [ ] **`DataframeConverter` log messages use Java string concatenation** — `DataframeConverter.groovy:155`, `:284`
  Replace with GString interpolation per project convention.

- [ ] **`powerTransform` parameter should be `Number`, not `double`** — `SmileFeatures.groovy:235`
  Per AGENTS.md, API parameters should accept `Number` and coerce internally so callers can pass `2` or `0.5` without explicit casts.

- [ ] **Correlation method should be an enum, not a `String`** — `SmileStats.groovy:520,559,598`
  The three magic values `"pearson"`, `"spearman"`, `"kendall"` should be an enum (e.g. `CorrelationMethod`) for self-documentation and compile-time safety.

- [ ] **`codenarc { ignoreFailures = true }` hides violations** — `build.gradle:15-16`
  Either fix the outstanding violations and remove the flag, or list specific suppressions. A blanket ignore lets style regressions accumulate silently.

### Documentation

- [ ] **README references stale versions** — `README.md:15-17`
  Update `matrix-smile:0.1.0` → `0.1.1`, `groovy:5.0.4` → current, `matrix-core:3.6.0` → current.

## Notes

- The module interfaces with Smile ML library (version 4.4.2), which uses primitive doubles for performance
- Some Java Math usage and `.doubleValue()` calls are justified for ML performance
- Module is constrained to Java 21 maximum due to Smile 4.x compatibility
- **Groovy 5.0+ is required**
  - Modern switch expression syntax (arrow operators) used in DataframeConverter.groovy requires Groovy 5.0+
  - This aligns with other matrix modules that use modern arrow-style switch statements
  - Project uses Groovy 5.0.4 in dependencies.gradle, uses `compileOnly` scope for flexibility
