# Matrix-ARFF v0.2.0 Plan

## Scope

This plan breaks the `matrix-arff` usability and configurability work into PR-sized sections. Each numbered section is intended to be implementable and reviewable as a separate pull request.

A task is only complete when its tests have passed and the exact test command has been recorded in the section or PR description.

## 1. Sparse ARFF Read Interoperability

1.1 [x] Extend `MatrixArffReader` to detect sparse data rows such as `{0 1.2, 3 'yes'}` while preserving the current dense row parsing path.

1.2 [x] Implement sparse row expansion against the declared attribute list so omitted values become `null` and index handling is validated.

1.3 [x] Define and document sparse read edge cases:
unsupported or invalid indices, duplicate indices in a row, sparse rows with quoted strings, and sparse rows containing missing values.

1.4 [x] Add tests in `matrix-arff/src/test/groovy/test/alipsa/matrix/arff/MatrixArffTest.groovy` for sparse numeric, nominal, string, and mixed rows.

1.5 [x] Update `matrix-arff/README.md` and `docs/tutorial/16-matrix-arff.md` with at least one sparse ARFF example and the supported scope for v0.2.0.

1.6 [x] Run tests and record the command: `./gradlew :matrix-arff:test`

## 2. Writer Schema and Type Configuration

2.1 [x] Expand `ArffWriteOptions` beyond `nominalMappings` with typed configuration for schema control, including:
`inferNominals`, `nominalThreshold`, `nominalColumns`, `stringColumns`, `attributeTypesByColumn`, `dateFormat`, and `dateFormatsByColumn`.

2.2 [x] Refactor `MatrixArffWriter` so attribute declaration generation goes through one typed configuration path instead of splitting logic across special-case overloads.

2.3 [x] Allow callers to force per-column ARFF output types where appropriate, including `STRING`, `NOMINAL`, `DATE`, `NUMERIC`, and `INTEGER`.

2.4 [x] Preserve explicit nominal value ordering when provided, and validate option conflicts with clear `IllegalArgumentException` messages.

2.5 [x] Add tests covering:
disabled nominal inference, custom nominal thresholds, forced string output, forced nominal output, global date format, and per-column date formats.

2.6 [x] Update `matrix-arff/README.md` with direct API and SPI examples showing how to control schema generation explicitly.

2.7 [x] Run tests and record the command: `./gradlew :matrix-arff:test`

## 3. Reader Strictness and Validation Modes

3.1 [ ] Expand `ArffReadOptions` with typed validation controls, starting with `strict`, `failOnUnknownAttributeType`, and `failOnRowLengthMismatch`.

3.2 [ ] In strict mode, fail fast for malformed rows, unknown attribute types, and other schema inconsistencies that are currently handled leniently.

3.3 [ ] Preserve the current lenient behavior as the default unless an explicit validation option is provided.

3.4 [ ] Improve parser error messages to include enough context for debugging, ideally including the offending line content and where practical the line number.

3.5 [ ] Add tests for strict vs lenient behavior covering:
extra row values, missing row values, unknown attribute types, and malformed sparse or dense rows.

3.6 [ ] Update `matrix-arff/README.md` and `docs/tutorial/16-matrix-arff.md` to document default behavior and when strict mode should be used.

3.7 [ ] Run tests and record the command: `./gradlew :matrix-arff:test`

## 4. Typed Options-First API Ergonomics

4.1 [ ] Add typed overloads for direct read operations, for example `MatrixArffReader.read(File, ArffReadOptions)` and matching overloads for `Path`, `URL`, `InputStream`, `Reader`, and `String`.

4.2 [ ] Add typed overloads for direct write operations, for example `MatrixArffWriter.write(Matrix, File, ArffWriteOptions)` and matching overloads for `Path`, `Writer`, `OutputStream`, and `writeString`.

4.3 [ ] Update `ArffFormatProvider` to delegate through the typed direct API so SPI usage and direct usage stay behaviorally aligned.

4.4 [ ] Keep the current convenience overloads for compatibility, but route them internally through the typed options path to reduce duplication.

4.5 [ ] Add tests covering the new typed overloads and verify that direct API calls and `Matrix.read(...)` or `matrix.write(...)` produce the same results.

4.6 [ ] Update `matrix-arff/README.md` to make the typed options API the primary documented path, with the raw convenience overloads shown as shortcuts.

4.7 [ ] Run tests and record the command: `./gradlew :matrix-arff:test`

## 5. Documentation and Discoverability

5.1 [ ] Refresh `matrix-arff/README.md` so the module’s current API surface is easy to discover without reading source code.

5.2 [ ] Document the default behaviors that are currently implicit:
default matrix naming, missing value handling, nominal inference heuristics, date formatting defaults, and SPI option discovery through `Matrix.listReadOptions(...)` and `Matrix.listWriteOptions(...)`.

5.3 [ ] Expand `docs/tutorial/16-matrix-arff.md` with focused examples for:
sparse input, explicit schema control, strict mode, and round-tripping with the generic Matrix SPI.

5.4 [ ] Update `matrix-arff/release.md` to reflect any user-visible changes delivered by sections 1 through 4.

5.5 [ ] Run tests and record the command: `./gradlew :matrix-arff:test`

## Recommended Merge Order

1. Section 1: sparse ARFF read interoperability
2. Section 2: writer schema and type configuration
3. Section 3: reader strictness and validation modes
4. Section 4: typed options-first API ergonomics
5. Section 5: documentation and discoverability
