# Matrix Stats v2.4.0 Release Roadmap

## Scope

This plan covers code quality, usability, commons-math3 removal, and new feature work for matrix-stats v2.4.0. Sections 1 through 8 address code health and should be completed before the feature sections. Each numbered section is intended to be implementable and reviewable as a separate pull request.

A task is only complete when its tests have passed and the exact test command has been recorded in the section or PR description.

**Usability and correctness issues:**

- 12 main source files missing `@CompileStatic`: Anova, Ellipse, Normalize, Randomize, Sampler, GoalSeek, KMeans, ClusteredPoint, GroupEstimator, KernelDensity, Kernel, BandwidthSelector.
- Duplicated utility methods across timeseries classes: `solveLinearSystem` (4 copies), `invertMatrix` (2 copies), `fitOLS` (2 copies), `calculateRSS` (2 copies).

- Bug risks: DurbinWatson division by zero when residuals are all zero; Accuracy.evaluatePredictions and `mape()` both divide by zero when actuals contain zero values; Ellipse silent failure returns empty data; ShapiroWilk `log(1-w)` when w=1; Sampler allows ratio=0.
- README/tutorial docs are stale beyond versions: tutorial imports outdated `se.alipsa.matrix.stats.Student`, paired t-test examples no longer match the API, and README links still point at the old standalone repository.
- README/tutorial dependency snippets use stale version numbers instead of BOM-based declarations.
- Empty cookbook page.
- POM metadata URLs use incorrect mono-repo paths (same issue fixed in matrix-csv).

## 1. CodeNarc P2 Fixes — Mechanical (Main Sources)

Bulk fixes that do not change behavior: constant extraction, brace insertion, and naming.

1.1 [x] Extract repeated numeric literals as named constants to fix DuplicateNumberLiteral (588 violations). Use `@SuppressWarnings('DuplicateNumberLiteral')` where the literal is a mathematical constant used inline (e.g. `0.0G` accumulators, exponents like `2`) and extracting it would reduce readability.

1.2 [x] Extract repeated string literals as named constants to fix DuplicateStringLiteral (166 violations). Use `@SuppressWarnings('DuplicateStringLiteral')` where the string is a short inline value used in result maps or toString methods.

1.3 [x] Add braces to all single-line `if`, `else`, and `for` statements to fix IfStatementBraces (85), ElseBlockBraces (10), and ForStatementBraces (2).

1.4 [x] Rename or suppress VariableName (42) and ParameterName (17) violations. Statistical convention names like `N`, `X`, `RSS`, `SSE` are intentional — add `@SuppressWarnings('VariableName')` or `@SuppressWarnings('ParameterName')` at the method or class level where the name follows standard statistical notation. Rename any that are not standard notation.

1.5 [x] Fix FieldName (4), PropertyName (3), and MethodName (2) violations. For KMeansPlusPlus `L1`/`L2`, either rename to `manhattanDistance`/`euclideanDistance` or suppress with `@SuppressWarnings('MethodName')` and a comment explaining they are standard metric names.

1.6 [x] Run tests and CodeNarc: `./gradlew :matrix-stats:spotlessApply :matrix-stats:test :matrix-stats:codenarcMain -g ./.gradle-user`

## 6. Bug Fixes and Input Validation

6.1 [x] DurbinWatson: add a guard against division by zero when all residuals are zero. Throw `IllegalArgumentException` with a descriptive message.

6.2 [x] Accuracy: fix zero-actual handling consistently in both `evaluatePredictions()` and `mape()`. Today both methods divide by zero when `actuals` contains `0`. Decide on one policy (for example skip zero actuals, reject the input, or return null/NaN-equivalent), implement it in both methods, and document the behavior.

6.3 [x] Ellipse: replace the silent empty-data return when input has fewer than 3 points or mismatched sizes with an `IllegalArgumentException`. Silent failures hide bugs in caller code.

6.4 [x] ShapiroWilk: guard against `Math.log(1.0 - w)` when `w == 1.0` (perfect normality), which produces `-Infinity`. Return a meaningful result or clamp `w` to `1.0 - epsilon`.

6.5 [x] Sampler: reject `ratio == 0.0` (currently only rejects negative and > 1.0). A zero ratio produces an empty training set.

6.6 [x] Correlation: add a regression test around Kendall comparison/tie handling with mixed numeric inputs. The regression matched the commons-math3 reference, so `compare()` was left unchanged.

6.7 [x] Run tests and verify bug fixes:

## 13. Formula Grammar and Parsing

13.1 [x] Define formula grammar and public API
- Supported syntax: response ~ terms, +, -, :, *, intercept control (0/1/-1)
- R-style operators: dot expansion (.), interaction expansion (^) for non-dot term sets, nesting (/)
- Limitation: power expressions that still contain dot expansion (for example `.^2` or `(. + x)^2`) are parsed, but normalization is deferred to model-frame evaluation in section 14 and currently rejected with a clear parse error.
- Parentheses grouping and backtick-quoted column names
- Built-in transforms: log(), sqrt(), exp(), poly(x, n), I(...)
- Multi-response policy: explicitly rejected for now during normalization with a clear parse error.

13.2 [x] Implement tokenizer and parser to AST
- Token types for operators, identifiers, function calls, literals, backticks
- AST nodes for terms, transforms, interactions, and grouping
- Clear error messages with location context
