# Section 17 Implementation Plan

This plan covers roadmap section 17, using the current formula/parser, model-frame, design-matrix, and fit-method implementation from sections 13 through 16 as the baseline. The focus here is not new formula features, but closing the validation gap around parser coverage, model-frame behavior, design-matrix stability, method-level regression tests, and user-facing documentation.

## 17. Formula Tests, Docs, and Validation

17.1 [x] Parser and AST tests

17.1.1 [x] Audit current parser coverage in `matrix-stats/src/test/groovy/formula/FormulaTest.groovy` and identify the remaining gaps against section 17.1:
- `.` in grouped expressions and updates
- `^` with grouped predictors and rejection edge cases
- `/` nesting combinations
- parentheses precedence across `+`, `:`, `*`, `/`, `^`
- backtick-heavy formulas in both response and predictors
- `I(...)`, `poly(...)`, and `s(...)` in valid and invalid positions

17.1.2 [x] Extend `matrix-stats/src/test/groovy/formula/FormulaTest.groovy` with parser-only cases for the remaining valid formulas:
- grouped dot expansion inputs that should normalize cleanly
- nested `/` combinations that should normalize to the expected interaction terms
- precedence-sensitive formulas that verify the exact normalized AST/string form
- backtick identifiers combined with transforms and interactions
- additive `s(x)` and `s(x, df)` formulas that should parse and normalize without changing intent

17.1.3 [x] Add invalid-formula regression tests in `matrix-stats/src/test/groovy/formula/FormulaTest.groovy` for:
- malformed `I(...)`, `poly(...)`, and `s(...)` argument lists
- unsupported multi-response forms
- invalid operator placement around `.`, `^`, `/`, and parentheses
- unterminated backticks and unmatched parentheses with position-aware errors

17.1.4 [x] Add AST-shape assertions where string normalization alone is too weak, so parser regressions in `FormulaExpression` and `ParsedFormula` are caught structurally rather than only textually.

17.2.4 [x] Add `subset` and `na.action` interaction tests in `matrix-stats/src/test/groovy/formula/FormulaModelFrameTest.groovy`:

- `NaAction.FAIL` rejects rows with nulls in referenced formula columns

17.2.5 [x] Add dot-expansion regression tests in `matrix-stats/src/test/groovy/formula/FormulaModelFrameTest.groovy` for:
