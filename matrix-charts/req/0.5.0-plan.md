# matrix-charts 0.5.0 Plan

Work is organized into phases. Each phase is one PR, merged in order.

---

## Phase 1 — Cleanup: dead code, docs, and small fixes

Low-risk removals, doc fixes, and one-line consistency fixes with no API changes.

- **1. Delete dead `ScaleModel`.**
  `render/ScaleModel.groovy` is `@Deprecated` with zero references in main or test code.
  Delete the file.

- **3. Remove or implement `AreaChart.create(…groupCol)`.**
  `AreaChart.groovy:46` throws `IllegalStateException('Not yet implemented')`.
  Remove the method (it is unused) and delete the `TODO` comment.

- **10. Make SVG output format consistent.**
  `ChartToSvg.writeSvg()` (lines 143, 149, 154) calls `svg.toXml()`.
  `Chart.writeTo()` uses `SvgWriter.toXmlPretty()`.
  Align `ChartToSvg` to use `SvgWriter.toXmlPretty()` as well.

- **D. Document `Chart.render()` default dimensions.**
  Add a note to the GroovyDoc on `Chart.render()` (line 331) stating that it renders at
  800x600 (the `RenderConfig` defaults).

---

## Phase 2 — Export bug fixes and consistency

Fix animation-stripping bugs and align export behaviour across all exporters.

- **2. Strip animation CSS in `ChartToImage.export()`.**
  The `BufferedImage` path goes through `SvgRenderer.toBufferedImage()` without stripping
  animation CSS. Add `AnimationCssStripper` to this path, matching `ChartToPng` behaviour.

- **7. Simplify `ChartToJpeg` animation stripping.**
  Remove the conditional `hasActiveAnimation()` check in the `export(CharmChart, …)`
  overloads. Always strip animation CSS, matching the `ChartToPng` approach.

---

## Phase 3 — Type safety: replace `Object` parameters with typed overloads

Addresses AGENTS.md "Avoid Object Parameters" rule. Pure API surface change, no
behaviour change.

- **8. Replace `Object` parameters in `ScaleDsl`, `GuidesDsl`, and `FacetDsl`.**

  **ScaleDsl** (`PlotSpec` lines 648-675): replace `setX(Object)`, `setY(Object)`,
  `setColor(Object)`, `setFill(Object)` with typed overloads:
  - `void setX(Scale value)`
  - `void setX(ScaleTransform value)`
  - `void setX(String value)`
  (same for y, color, fill)

  **GuidesDsl** (`PlotSpec` lines 1210-1228): replace all seven `Object` setters with:
  - `void setColor(GuideSpec value)`
  - `void setColor(GuideType value)`
  - `void setColor(String value)`
  - `void setColor(boolean value)` (false -> `GuideSpec.none()`)
  (same for fill, size, shape, alpha, x, y)

  **FacetDsl** (`PlotSpec` line 1133): replace `setLabeller(Object)` with
  `setLabeller(Labeller)`.

- **11. Type `WrapDsl` fields.**
  Change `List vars = []` to `List<String> vars = []` and `Object labeller` to
  `Labeller labeller` in `FacetDsl.WrapDsl` (`PlotSpec` line 1182).

---

## Phase 4 — Export: `writeTo()` extension dispatch and `PlotGrid` support

Unified export convenience for both `Chart` and `PlotGrid`.

- **5. Make `writeTo()` auto-detect format from file extension.**
  `Chart.writeTo(File)` should inspect the extension and dispatch:
  - `.png` -> `ChartToPng`
  - `.jpg` / `.jpeg` -> `ChartToJpeg`
  - anything else (including `.svg`) -> SVG (current behaviour)

  Add `writeTo(String)` and `writeTo(File)` to `PlotGrid` with the same logic.
  Extract extension-detection into a shared utility to avoid duplication.

- **6. Add `PlotGrid` overloads to export classes.**
  Add `export(PlotGrid, …)` to `ChartToPng`, `ChartToJpeg`, `ChartToSvg`,
  `ChartToSwing`, `ChartToJfx`, and `ChartToImage`. Each delegates through
  `grid.render()` to the existing `Svg` overload.

## Phase 5 — Migrate deprecated `Plot` usage out of tests

- **9. Replace `Plot` calls in tests with current export API.**
  Migrate `PngTest`, `LineChartTest`, `BoxChartTest`, and `PlotCompatibilityTest`
  from the `@Deprecated` `Plot` class to `ChartToPng`, `ChartToImage`, etc.
  After this phase, `Plot` has zero test references and can be removed in a future
  release.

## Phase 6 — DSL ergonomics

New DSL surface area. Each item is independently useful but grouped to keep the PR
focused on usability.

- **A. Expose theme presets in `ThemeDsl`.**
  Add factory methods or a `preset` setter to `ThemeDsl` so users can write
  `theme { preset = 'minimal' }` or `theme { use minimal() }` without importing
  `CharmThemes` directly.

- **B. Add `coordFlip()` shorthand to `PlotSpec`.**
  Add `PlotSpec coordFlip()` that sets `coord.type = CharmCoordType.FLIP`, mirroring
  ggplot2's `coord_flip()`.

- **4. Expose `CssAttributesSpec` in the DSL.**
  Add a `cssAttributes {}` closure method to `PlotSpec` and wire it through to
  `PlotSpec.build()` (currently always passes `null`).

## Phase 7 — `+` operator composition (optional)

Larger API addition, can be deferred if closure DSL is deemed sufficient.

- **C. Add `plus()` operator to `PlotSpec`.**
  Implement `PlotSpec plus(LayerBuilder)`, `PlotSpec plus(Theme)`,
