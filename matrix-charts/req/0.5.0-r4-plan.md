## Task 5: Combine double-pass in `trainScales`

**Background:** `trainScales` (CharmRenderer lines 97-147) iterates all layers twice — once to collect values for global scale training, once for per-layer scale training. Could be combined into a single pass.

**Files:**
- Modify: `matrix-charts/src/main/groovy/se/alipsa/matrix/charm/CharmRenderer.groovy`

- [ ] **Step 1: Refactor `trainScales` to single pass**

  Collect both global values and per-layer data in one iteration:
  ```groovy
  private void trainScales(RenderContext context) {
    List<Object> xValues = []
    List<Object> yValues = []
    List<Object> colorValues = []
    List<Object> fillValues = []
    List<Object> sizeValues = []
    List<Object> shapeValues = []
    List<Object> alphaValues = []
    List<Object> linetypeValues = []

    // Single pass: collect global values + per-layer data
    context.chart.layers.eachWithIndex { LayerSpec layer, int layerIdx ->
      Matrix sourceData = resolveLayerData(context.chart.data, layer)
      List<Integer> rowIndexes = defaultRowIndexes(sourceData?.rowCount() ?: 0)
      Mapping mapping = effectiveMapping(context.chart.mapping, layer)
      List<LayerData> pipelineData = runPipeline(context, layer, sourceData, mapping, rowIndexes)

      // Accumulate global values
      xValues.addAll(pipelineData.collect { it.x })
      yValues.addAll(pipelineData.collect { it.y })
      colorValues.addAll(pipelineData.collect { it.color })
      fillValues.addAll(pipelineData.collect { it.fill })
      sizeValues.addAll(pipelineData.collect { it.size })
      shapeValues.addAll(pipelineData.collect { it.shape })
      alphaValues.addAll(pipelineData.collect { it.alpha })
      linetypeValues.addAll(pipelineData.collect { it.linetype })

      // Per-layer scale training (if applicable)
      if (layer.scales != null && !layer.scales.isEmpty()) {
        TrainedScales layerTrained = ScaleEngine.trainLayerScales(
            layer.scales, context.config, pipelineData, context.chart)
        context.layerScales[layerIdx] = layerTrained
        checkScaleDivergence(layerIdx, layerTrained, context)
      }
    }

    // Global scale training (after collecting all values)
    TrainedScales trained = ScaleEngine.train(
        context.chart, context.config, xValues, yValues, colorValues, fillValues,
        sizeValues, shapeValues, alphaValues, linetypeValues)
    context.xScale = trained.x
    context.yScale = trained.y
    context.colorScale = trained.color
    context.fillScale = trained.fill
    context.sizeScale = trained.size
    context.shapeScale = trained.shape
    context.alphaScale = trained.alpha
    context.linetypeScale = trained.linetype
  }
  ```

  ```groovy
  private static Mapping effectiveMapping(Mapping plotMapping, LayerSpec layer) {
    if (!layer.inheritMapping) {
      return new Mapping()
    }
    if (layer.mapping == null || layer.mapping.isEmpty()) {
      return plotMapping  // no copy needed — plotMapping is already immutable after build()
    }
    Mapping mapping = plotMapping.copy()
    mapping.apply(layer.mapping.mappings())
    mapping
  }
  ```

## Task 12: Extend `deepFreezeValue` for `Collection` and `NavigableSet`

**Background:** `deepFreezeValue` (Chart line 236-256) handles `Map`, `List`, `Set` but not `Collection` or `Iterable` supertypes. A `SortedSet` or `NavigableSet` would fall through to the identity return.

**Files:**
- Modify: `matrix-charts/src/main/groovy/se/alipsa/matrix/charm/Chart.groovy`

- [ ] **Step 1: Add Collection and NavigableSet handling**

  ```groovy
  private static Object deepFreezeValue(Object value) {
    if (value == null) return null
    if (value instanceof Map) {
      Map<Object, Object> map = (Map<Object, Object>) value
      Map<Object, Object> frozen = new LinkedHashMap<>()
      map.each { k, v -> frozen[k] = deepFreezeValue(v) }
      return Collections.unmodifiableMap(frozen)
    }
    if (value instanceof Collection) {
      Collection<Object> coll = (Collection<Object>) value
      List<Object> frozen = new ArrayList<>(coll.size())
      coll.each { v -> frozen << deepFreezeValue(v) }
      return Collections.unmodifiableList(frozen)
    }
    if (value instanceof Set) {
      Set<Object> set = (Set<Object>) value
      Set<Object> frozen = new LinkedHashSet<>(set)
      frozen.replaceAll { deepFreezeValue(it) }
      return Collections.unmodifiableSet(frozen)
    }
    if (value instanceof NavigableSet) {
      NavigableSet<Object> nav = (NavigableSet<Object>) value
      Set<Object> frozen = new TreeSet<>(nav.comparator())
      nav.each { v -> frozen << deepFreezeValue(v) }
      return Collections.unmodifiableNavigableSet(frozen)
    }
    value
  }
  ```

  **Note:** The `Collection` check must come before `Set` since `Set` extends `Collection`. The order is: `Map` → `Collection` (catches `List`, `Set`, `NavigableSet`) → `Set` (redundant now but kept for clarity) → `NavigableSet`.

  Actually, since `Collection` catches `Set` and `NavigableSet`, we can simplify:
  ```groovy
  if (value instanceof Collection) {
    Collection<Object> coll = (Collection<Object>) value
    List<Object> frozen = new ArrayList<>(coll.size())
    coll.each { v -> frozen << deepFreezeValue(v) }
    return Collections.unmodifiableList(frozen)
  }
  ```

  This handles `List`, `Set`, `NavigableSet`, `SortedSet`, etc. uniformly — all become unmodifiable lists after deep-freezing, which is the correct behavior since the original collection type is not preserved in the frozen output anyway.
