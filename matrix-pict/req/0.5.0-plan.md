## Global Constraints

- Groovy 5.0.6, Java 21 (`options.release = 21`)
- `@CompileStatic` is enforced project-wide via `config/groovy/compileStatic.groovy` for all production source; use `@CompileDynamic` only when static compilation would be significantly more complex
- All public classes and public methods require GroovyDoc
- JavaBean getter naming: add alias getters (`getXAxisTitle()` etc.) for new public API; **do not remove existing lowercase getters** (`getxAxisTitle()` etc.) because `Introspector.decapitalize` keeps `XAxisTitle` uppercased, so removing them would break `chart.xAxisTitle` Groovy property access in existing callers

    }
    Matrix firstTable = series[0]
    if (firstTable.columnCount() != REQUIRED_COLUMN_COUNT) {

            throw new IllegalArgumentException("Column mismatch in series $idx. First series has type $firstColumn.name " +

        }
        if (DataType.differs(secondColumn, col1Type)) {
            throw new IllegalArgumentException("Column mismatch in series $idx. First series has type $secondColumn.name " +
                    "in the second column but this series has $col1Type")
        }
    }
}

## Task 3: Fix getter naming inconsistency in Chart.groovy (Issue #3)

**Issue #3:** Four getter methods in `Chart.groovy` violate JavaBean naming convention: `getxAxisTitle()`, `getyAxisTitle()`, `getxAxisScale()`, `getyAxisScale()`. The fix is to **add** properly-named alias getters — do not remove the existing lowercase-named methods.

**Why keep the lowercase getters?** `Introspector.decapitalize("XAxisTitle")` returns `XAxisTitle` (unchanged) because both the first *and* second characters are uppercase, so `getXAxisTitle()` would expose Groovy property `chart.XAxisTitle`, not `chart.xAxisTitle`. Existing callers using `chart.xAxisTitle` rely on `getxAxisTitle()` for property resolution. Removing it would silently break those callers. The safe approach: add the JavaBean-style aliases alongside the existing getters, and optionally annotate the old ones `@Deprecated`.

**Files:**
- Modify: `matrix-pict/src/main/groovy/se/alipsa/matrix/pict/Chart.groovy` (four getter methods)
- Test: `matrix-pict/src/test/groovy/chart/ChartFactoryBaselineTest.groovy`

- `create(String title, Matrix data, String categoryColumnName, String valueColumn)` — splits the matrix by categoryColumnName and creates per-group box series

Add two private static methods above the first `create()` and redirect both `create()` overloads and `Builder.build()` to use them:

```groovy
private static BoxChart fromCategoryValue(String title, Matrix data, String categoryCol, String valueCol) {
    Map<String, Matrix> groups = data.split(categoryCol).sort() as Map<String, Matrix>
    BoxChart chart = new BoxChart()
    chart.title = title
    chart.categorySeries = ListConverter.toStrings(groups.keySet()) as List<?>
    chart.valueSeries = groups.values().collect { Matrix m -> m[valueCol].removeNulls() as List<?> }
    chart.valueSeriesNames = chart.categorySeries as List<String>
    chart.xAxisTitle = categoryCol
    chart.yAxisTitle = valueCol
    chart
}

private static BoxChart fromColumns(String title, Matrix data, List<String> columnNames) {
    BoxChart chart = new BoxChart()
    chart.title = title
    chart.categorySeries = columnNames as List<?>
    chart.valueSeries = columnNames.collect { String col -> data[col] as List<?> }
    chart.valueSeriesNames = columnNames
    chart.xAxisTitle = 'X'
    chart.yAxisTitle = 'Y'
    chart
}
```
