# Matrix Core v3.8.0 Release Roadmap

## 1. Align Column scalar-null arithmetic with element-wise null propagation

### Background

Column arithmetic already propagates null for individual elements within lists:

```groovy
new Column([1, null, 3]) + [10, 20, 30]  // → [11, null, 33]
new Column([1, 2, 3]) + [null, 20, 30]   // → [null, 22, 33]
```

However, the scalar overloads (`plus(Object)`, `minus(Object)`, `multiply(Number)`,
`div(Number)`, `power(Number)`) throw `IllegalArgumentException` when the scalar
argument is null:

```groovy
new Column([1, 2, 3]) + (Number) null  // → IllegalArgumentException
```

This is inconsistent. The private `applyOperation` helper already handles null
elements in `this` by propagating null, so the scalar-null case is the only gap.

### Required behavior

When a scalar argument is null, return a Column where every element is null,
consistent with the element-wise null propagation already in place.

A null `List` reference argument should continue to throw `IllegalArgumentException`
since that indicates a programming error, not missing data.

| Operation | Current | Required |
|---|---|---|
| `[1, 2, 3] + (Number) null` | `IllegalArgumentException` | `[null, null, null]` |
| `[1, null, 3] + 2` | `[3, null, 5]` | `[3, null, 5]` (unchanged) |
| `[1, 2, 3] + [null, 2, 3]` | `[null, 4, 6]` | `[null, 4, 6]` (unchanged) |
| `[1, 2, 3] + (List) null` | `IllegalArgumentException` | `IllegalArgumentException` (unchanged) |

### Scope

Methods in `Column.groovy`:
- `plus(Object)`
- `minus(Object)`
- `multiply(Number)`
- `div(Number)`
- `power(Number)`

### Rationale

This is an alignment fix: the List overloads already propagate null for individual
elements, and `matrix-tablesaw` 0.3.0 `BigDecimalColumn` arithmetic propagates nulls
throughout. Making scalar-null consistent closes the last gap without introducing
a new behavioral contract.

### Tests

- `column + (Number) null` returns all-null Column
- `column - (Number) null` returns all-null Column
- `column * (Number) null` returns all-null Column
- `column / (Number) null` returns all-null Column
- `column ** (Number) null` returns all-null Column
- Existing List-null-reference tests still throw `IllegalArgumentException`
- Existing element-wise null propagation tests remain unchanged
