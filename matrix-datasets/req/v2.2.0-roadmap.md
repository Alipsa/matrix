# Bugs / Issues

1. [x] `Rdatasets.groovy:23` does network I/O in a static initializer. Any first touch of `Rdatasets`, even
   invalid-argument calls, downloads `datasets.csv`. This makes class loading fail offline and can slow or hang
   apps. Prefer lazy cached loading in `overview()`, with a clear error message and possibly a `refresh()` method.

2. [x] `build.gradle:35` uses `compileOnly` for runtime-essential / public API dependencies. Decision: keep compileOnly for BOM flexibility. `Matrix` is returned
   from public methods, and GINQ is used by `Rdatasets`; consumers currently need to know to add `matrix-core`
   and `groovy-ginq` manually. `matrix-core` should be `api` (transitive) and `groovy-ginq` should be
   `implementation`.

3. [x] `DatasetTest.groovy:238` (`testFromUrl`) and `RdatasetsTest.groovy:14-36` (`testOverview`, `fetchInfo`,
   `testFetchData`) run external network calls unconditionally. These should be gated behind the repo's
   `RUN_EXTERNAL_TESTS` mechanism to avoid flaky CI and offline failures.

4. [x] `Dataset.groovy:79` — `iris()` converts `Id: Integer` but `iris.csv` has no `Id` column (it has exactly
   five columns: `Sepal Length`, `Sepal Width`, `Petal Length`, `Petal Width`, `Species`). The phantom conversion
   silently does nothing but is misleading. Remove `Id: Integer` from the `convert()` call and remove the `Id`
   line from `descIris()`. Tests already assert 5 columns so no test change is needed.

5. [x] `FileUtil.groovy:106` — `getResourcePath()` calls `url.getFile()` without a null guard. When
   `getResourceUrl()` returns `null` (resource not found) this throws an undocumented `NullPointerException`
   instead of the `FileNotFoundException` documented on the related APIs. Add: `if (url == null) throw new
   FileNotFoundException("Resource not found: $name")`.

6. [x] `Dataset.groovy:455-469` — `descDiamonds()` omits the `depth` and `table` variables from its listing,
   even though `diamonds()` converts both columns and they appear in the CSV header
   (`carat, cut, color, clarity, depth, table, price, x, y, z`). Add descriptions for both:
   - `depth`: total depth percentage = z / mean(x, y)
   - `table`: width of top of diamond relative to widest point

7. [x] `README.md:11,29` — Setup examples list Groovy `5.0.6` but the project uses `5.0.5`. Verified: README and libs.versions.toml both agree on 5.0.6.
   (`gradle/libs.versions.toml`). Update to match the actual version.

8. [x] `Dataset.groovy:165,184` — `mpg()` and `diamonds()` begin with an explicit `return` statement on their
   last (and only) expression. Per project style, `return` is only used for early exits; implicit return should
   be used here.

9. [x] `Dataset.groovy:353,375,391,403,419,433,455,471` — All `descXxx()` methods use an explicit `return`
   before their heredoc string literal. These should all use implicit return.

10. [x] `Dataset.groovy:312-325` — `describe(String)` uses 10 sequential `if` comparisons. Replace with an
    idiomatic Groovy map dispatch:
    ```groovy
    private static final Map<String, Closure<String>> DESCRIBERS = [
        airquality  : { descAirquality() },
        cars        : { descCars() },
        mtcars      : { descMtcars() },
        iris        : { descIris() },
        plantgrowth : { descPlantGrowth() },
        toothgrowth : { descToothGrowth() },
        usarrests   : { descUsArrests() },
        mpg         : { descMpg() },
        diamonds    : { descDiamonds() },
        mapdata     : { descMapData() },
        map_data    : { descMapData() }
    ]
    static String describe(String tableName) {
        DESCRIBERS[tableName.toLowerCase()]?.call() ?: "Unknown table: ${tableName}"
    }
    ```

# Usability Enhancements

11. [x] Add discoverability helpers. Right now users must know method names upfront:
    - `Dataset.names()` — returns a sorted `List<String>` of built-in dataset names
    - `Dataset.mapNames()` — returns the valid map dataset names
    - `Dataset.load(String name)` — dispatches to the appropriate loader by name (useful for REPL / scripting)

12. [x] Improve `mapData()` error message ergonomics:
    - Include the list of valid names in the `IllegalArgumentException`: `"no map data exists for '$datasetName'. Valid names: ${MAP_DATA_FILES.keySet().sort().join(', ')}"`
    - Trim whitespace from `datasetName` before lowercasing (guard against accidental leading/trailing spaces)
    - Consider adding `Dataset.mapRegions(String datasetName)` to list distinct region values for a given map dataset

13. [x] Make `Rdatasets.fetchData()` easier to use:
    - Add a single-argument overload `fetchData(String packageSlashItem)` that splits on `/`
      (e.g., `fetchData('datasets/iris')`)
    - Add `Rdatasets.search(String text)` that filters the overview by title/item for quick lookup
    - Consider an optional type-inference pass that converts obviously numeric columns to `BigDecimal`
      and integer-valued columns to `Integer`, so callers don't always have to call `convert()` manually
